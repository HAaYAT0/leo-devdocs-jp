Aleo ブロックチェーンエコシステムのネットワークアーキテクチャは、コンセンサスネットワークとピアツーピア（P2P）ネットワークの 2 つの構成要素で成り立っています。この枠組みの中で、バリデータ、プローバー、クライアントという 3 種類の基本的なノードタイプが存在します。

### Validator

[**バリデータ**](./validators.md)はコンセンサスネットワークと P2P ネットワークの両方にとって不可欠な存在です。コンセンサスネットワークでは通常ポート 5000、P2P ネットワークではポート 4130 を利用します。バリデータ同士はポート 5000 を介して相互接続され、一体となったコンセンサスネットワークを形成します。重要な点として、クライアントとプローバーはコンセンサスネットワークへのアクセスが**制限**されています。

バリデータノードの主な役割は AleoBFT のルールに従い、新しいブロックの生成を円滑に進めることです。

Aleo の P2P ネットワークでは、クライアントとプローバーが P2P メカニズムを通じて相互接続します。加えて、指定されたバリデータノード群とも接続し、コンセンサスネットワークから最新ブロックを取得します。

### Client

[**クライアント**](./client.md)は、コンセンサスネットワークで生成されたブロックを同期し、台帳を更新することでエコシステムを支えます。クライアントが提供する RPC を通じて、ユーザーは Aleo ネットワークのリアルタイムな台帳状態にアクセスし、トランザクションをネットワークへブロードキャストできます。新しいブロックへトランザクションが取り込まれると、それがネットワーク上で正常に実行されたことを意味します。

### Prover

[**プローバー**](./provers.md) ノードは、コンセンサスネットワークで生成された CoinbasePuzzle を同期する役割を担います。指定された条件を満たす解を導くために必要なアルゴリズムを実行し、得られた解を P2P ネットワーク全体にブロードキャストしてコンセンサスネットワークへ送信します。コンセンサスネットワークがこれらの解を新しいブロックへ統合すると、プローバーは対応する CoinbaseReward を受け取る権利を得ます。
